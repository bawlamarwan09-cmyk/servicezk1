# Evolura Technical Services

Production website for Evolura Technical Services, a Dubai-based commercial
cleaning and technical maintenance company serving requests across the UAE.

## Stack

- Next.js 16 App Router
- React 19 and TypeScript
- Tailwind CSS utilities with project CSS in `app/globals.css`
- Native Next.js output for Vercel
- vinext output for OpenAI Sites/Cloudflare compatibility

## Local development

Use Node.js 22.

```bash
npm install
npm run dev
```

Open `http://localhost:3000`.

## Production URL and canonicals

Set one permanent production origin in the deployment environment:

```bash
NEXT_PUBLIC_SITE_URL=https://www.example.com
```

Use the origin only, without a path or trailing slash. This value controls
canonical URLs, Open Graph URLs, structured-data IDs, `robots.txt`, and
`sitemap.xml`. If it is not set on Vercel, the site uses
`VERCEL_PROJECT_PRODUCTION_URL` when available.

Do not use a temporary preview deployment URL as `NEXT_PUBLIC_SITE_URL`.

## Vercel deployment

The project must use the native Next.js build:

- Framework preset: Next.js
- Build command: `npm run build`
- Output directory: leave blank so Vercel uses `.next`
- Node.js: 22.x

Do not configure `dist` as the Vercel output directory. `dist` is generated by
the separate `build:sites` command and is not a Next.js `.next` deployment.

## OpenAI Sites build

The repository contains `.openai/hosting.json`, so the Sites-compatible build is:

```bash
npm run build:sites
```

## Content and route structure

- `/` — main landing page and quotation form
- `/services` — crawlable service directory and comparison
- `/services/[service]` — five focused service pages
- `/about` — factual company and service-process information
- `/contact` — consistent business details and the quotation form
- `/privacy` — quote-form and external-contact data flow
- `/manifest.webmanifest`, `/robots.txt`, and `/sitemap.xml` — generated metadata routes

Shared business details and contact URLs live in `app/site-config.ts`. Shared
service content, internal relationships, FAQs, and image metadata live in
`app/seo-content.ts`. Update these sources instead of duplicating information in
individual routes.

## Images

Service imagery uses matching 8:5 WebP assets:

- `1440 × 900` for the full source
- `720 × 450` for the responsive small source

Keep meaningful alternative text in `app/seo-content.ts`. Decorative interface
graphics should remain hidden from assistive technology. The homepage hero has
separate 960, 1440, and full-width WebP sources and must retain explicit width
and height attributes to prevent layout shift.

## Quote form behavior

The quotation form validates its fields in the browser and sends the request to
the same-origin `/api/contact` route. That server route forwards the request to
the private n8n contact workflow configured by `N8N_CONTACT_WEBHOOK_URL`.

## Customer review behavior

The homepage review form sends only to the same-origin `/api/reviews` route. The
browser never receives either n8n URL or its authentication token. Configure:

```bash
N8N_REVIEW_SUBMIT_URL=https://automation.example.com/webhook/website-review
N8N_REVIEW_FEED_URL=https://automation.example.com/webhook/website-reviews
N8N_REVIEW_WEBHOOK_SECRET=replace-with-a-long-random-secret
REVIEW_RATE_LIMIT_SECRET=replace-with-another-long-random-secret
```

The submit workflow must store every new review as pending. The feed workflow
must return only approved, consented reviews and must omit customer email,
internal status, reply content and other private fields. The public response
shape is:

```json
{
  "reviews": [
    {
      "name": "Customer display name",
      "rating": 5,
      "review": "Approved review text",
      "service": "commercial-office-cleaning-dubai",
      "createdAt": "2026-07-15T12:00:00.000Z"
    }
  ]
}
```

Use n8n Header Auth for both webhooks with an `Authorization` header whose
value is `Bearer <N8N_REVIEW_WEBHOOK_SECRET>`. Do not expose full customer email
addresses in the feed. Review and run `db/website-reviews-postgres.sql` before
updating the workflow queries. See `.env.example` for the complete non-secret
template.

An importable approved-review feed is provided at
`n8n/evolura-public-reviews.workflow.json`. After importing it, select the
Header Auth and least-privilege PostgreSQL credentials, publish the workflow,
and set `N8N_REVIEW_FEED_URL` to its production `/webhook/public-reviews` URL.
The n8n response is not cached. The website refreshes the feed every 30 seconds
while the review section is near the viewport and keeps a 15-second shared edge
cache to protect the workflow from duplicate traffic. New submissions remain
hidden until they are approved and have publication consent.

The in-process website limiter is only a first layer. Before launch, use the
PostgreSQL limiter table from the migration in the n8n submit workflow and run
its documented cleanup query on an hourly n8n schedule.

## Validation

```bash
npm run lint
npx tsc --noEmit
npm run build
npm test
```

structured data, service routes, crawl directives, form fields, and key
accessibility behavior.
