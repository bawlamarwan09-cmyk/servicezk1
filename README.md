# Evolura Technical Services

Production website for Evolura Technical Services, a Dubai-based commercial
cleaning and technical maintenance company serving requests across the UAE.

## Stack

- Next.js 16 App Router
- React 19 and TypeScript
- Tailwind CSS utilities with project CSS in `app/globals.css`
- Native Next.js output for Vercel
- vinext output for OpenAI Sites/Cloudflare compatibility

## Local development

Use Node.js 22.

```bash
npm install
npm run dev
```

Open `http://localhost:3000`.

## Production URL and canonicals

Set one permanent production origin in the deployment environment:

```bash
NEXT_PUBLIC_SITE_URL=https://www.example.com
```

Use the origin only, without a path or trailing slash. This value controls
canonical URLs, Open Graph URLs, structured-data IDs, `robots.txt`, and
`sitemap.xml`. If it is not set on Vercel, the site uses
`VERCEL_PROJECT_PRODUCTION_URL` when available.

Do not use a temporary preview deployment URL as `NEXT_PUBLIC_SITE_URL`.

## Vercel deployment

The project must use the native Next.js build:

- Framework preset: Next.js
- Build command: `npm run build`
- Output directory: leave blank so Vercel uses `.next`
- Node.js: 22.x

Do not configure `dist` as the Vercel output directory. `dist` is generated by
the separate `build:sites` command and is not a Next.js `.next` deployment.

## OpenAI Sites build

The repository contains `.openai/hosting.json`, so the Sites-compatible build is:

```bash
npm run build:sites
```

## Content and route structure

- `/` — main landing page and quotation form
- `/services` — crawlable service directory and comparison
- `/services/[service]` — five focused service pages
- `/about` — factual company and service-process information
- `/contact` — consistent business details and the quotation form
- `/privacy` — quote-form and external-contact data flow
- `/manifest.webmanifest`, `/robots.txt`, and `/sitemap.xml` — generated metadata routes

Shared business details and contact URLs live in `app/site-config.ts`. Shared
service content, internal relationships, FAQs, and image metadata live in
`app/seo-content.ts`. Update these sources instead of duplicating information in
individual routes.

## Images

Service imagery uses matching 8:5 WebP assets:

- `1440 × 900` for the full source
- `720 × 450` for the responsive small source

Keep meaningful alternative text in `app/seo-content.ts`. Decorative interface
graphics should remain hidden from assistive technology. The homepage hero has
separate 960, 1440, and full-width WebP sources and must retain explicit width
and height attributes to prevent layout shift.

## Quote form behavior

The interactive website does not submit quote-form data to an Evolura website
database. It validates five fields in the browser, passes them to WhatsApp to
prepare a message, and lets the visitor review that message before deciding
whether to send it. Before JavaScript is ready, the customer fields and submit
button remain disabled. Keep the disclosure in `app/QuoteRequestForm.tsx`
accurate if this data flow changes.

## Validation

```bash
npm run lint
npx tsc --noEmit
npm run build
npm test
```

`npm test` creates the Sites-compatible build and verifies rendered metadata,
structured data, service routes, crawl directives, form fields, and key
accessibility behavior.
